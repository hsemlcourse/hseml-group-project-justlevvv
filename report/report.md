## 1. Введение и постановка задачи

- **Цель проекта:** Предсказать, уволится ли сотрудник в ближайшие несколько месяцев на основе его профессиональных и личных характеристик (возраст, должность, отдел, уровень дохода и т.д.). Практическая ценность: получение информации о рисках увольнения заранее для кадровой политики.
- **Формулировка задачи:** классификация
- **Обоснование метрики качества:** В датасете есть сильный дисбаланс классов (80% остаются, 20% увольняются).
  Основное внимание уделяем F1-score для класса уволившихся (Yes), так как:
  - необходимо одновременно контролировать и Recall, и Precision;
  - Accuracy может вводить в заблуждение -- модель, предсказывающая исключительно "No", показывает Accuracy 0.8, но не несёт ценности в задаче.
  Дополнительно отслеживаем Recall (Yes) (долю найденных реальных увольнений) и Precision (Yes).

## 4. Baseline-модель

- **Модели:**
  - `DummyClassifier` (стратегия `most_frequent`) — всегда предсказывает большинство ("No"), служит абсолютным минимумом;
  - `LogisticRegression` с `class_weight='balanced'` — простая линейная модель, учитывающая дисбаланс классов.
- **Результаты на тестовой выборке:**

| Модель                 | Accuracy | Precision (Yes) | Recall (Yes) | F1-score (Yes) |
|------------------------|----------|-----------------|--------------|----------------|
| Dummy (most_frequent)  | 0.80     | 0.00            | 0.00         | 0.00           |
| Logistic Regression    | 0.51     | 0.20            | 0.48         | 0.28           |

- **Интерпретация:**
  - Dummy‑модель бесполезна для поиска увольнений.
  - Логистическая регрессия значительно превосходит случайное угадывание (F1=0.28 против ~0.20 у стратифицированного случайного предсказания). Она находит 48% реальных увольнений, но даёт много ложных тревог (Precision 0.20). Это ожидаемо для первой модели с приоритетом на полноту.
  - Accuracy 0.51 не должна рассматриваться как главная метрика из-за дисбаланса классов.

## 5. Эксперименты

Проведены эксперименты с несколькими моделями, подбором гиперпараметров и ансамблем.
Предположим, что более сложные нелинейные модели (Random Forest, XGBoost) смогут лучше улавливать слабые зависимости в данных.

**Модели, использованные в экспериментах:**
- Logistic Regression (baseline)
- KNN
- Decision Tree
- Random Forest
- XGBoost
- Linear SVC

Каждая модель обучалась с параметрами по умолчанию и сбалансированными весами классов (если применимо). Результаты на тестовой выборке:

| Модель           | Accuracy | Precision (Yes) | Recall (Yes) | F1 (Yes) | ROC-AUC | PR-AUC |
|------------------|----------|-----------------|--------------|----------|---------|--------|
| Logistic Regression | 0.51  | 0.20            | 0.48         | 0.28     | 0.51    | 0.21   |
| KNN              | 0.76     | 0.18            | 0.05         | 0.08     | 0.48    | 0.20   |
| Decision Tree    | 0.69     | 0.21            | 0.21         | 0.21     | 0.51    | 0.20   |
| Random Forest    | 0.80     | 0.00            | 0.00         | 0.00     | 0.50    | 0.20   |
| XGBoost          | 0.72     | 0.22            | 0.15         | 0.18     | 0.51    | 0.21   |
| Linear SVC       | **0.50** | 0.20            | **0.52**     | **0.29** | 0.49    | 0.19   |

**Наблюдения:**
- Все модели испытывают трудности из-за низкой информативности признаков (подтверждено EDA - корреляции с Attrition слабые).
- Random Forest, не предсказал ни одного увольнения (Recall=0). Вероятная причина - низкая корреляция признаков деревья стремятся к большинству.
- Линейные модели (Logistic Regression, Linear SVC) показали наилучший баланс между Precision и Recall, достигая F1 около 0.29.
- KNN и XGBoost продемонстрировали низкий Recall, что делает их непригодными для этой задачи.

**Подбор гиперпараметров:**
Лучшие модели по F1 (Linear SVC, Logistic Regression) были отобраны для тюнинга, но для них не задана сетка параметров в текущей реализации. Можно применить RandomizedSearchCV с более широким набором параметров.

**Ансамбль (Voting):**
Ансамбль из трёх лучших моделей (Logistic Regression, Decision Tree, Linear SVC) с мягким голосованием не помог (F1=0.21), что подтверждает, что модели замкнуты в своих ошибках из-за слабых признаков.

**Вывод по экспериментам:**
Качество предсказаний ограничено качеством датасета. Лучший результат - Linear SVC (F1=0.29), который способен обнаружить 52% уволившихся при точности 20%. Это лучше, чем случайное угадывание, но бессмысленно в практическом применении.

## 6. Финальная модель и интерпретируемость

**Финальная модель:** Linear SVC с `class_weight='balanced'`.  
Показала наивысший F1-score (0.29) на тестовой выборке среди всех протестированных моделей.

**Интерпретируемость:**
Linear SVC строит линейную разделяющую поверхность. Абсолютные значения коэффициентов позволяют судить о важности признаков. Топ-5 признаков с наибольшим влиянием:
- Department_Marketing
- Deparment_HR
- Job_Role_Manager
- Department_IT
- Deparment_Sales
(Полный результат приведён на плоте в ноутбуке)

**Ограничения:**
- Низкая общая предсказательная сила (F1=0.29) - следствие слабых корреляций в данных.
- Высокий Recall (52%) при низкой точности (20%) означает, что модель можно использовать для скрининга (например, для выделения группы риска), но не для точных кадровых решений.
- Необходимы более информативные признаки (например, данные о производительности, опросах удовлетворённости), чтобы повысить качество.
